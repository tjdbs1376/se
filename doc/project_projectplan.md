# 프로젝트 관리 계획서 
# 1. 서론 
## 1.1 문서 목적 및 범위
  본 프로젝트 계획서는 **클라우드 파일 공유 시스템 (미니 드라이브)** 개발에 대한 전반적인 내용을 정리하여 프로젝트를 진행하는 데 있어 지표로 삼는 것을 목적으로 한다.

## 1.2 프로젝트 개요 
### 1.2.1 프로젝트 정의 
조직 내부에서 파일을 중앙 관리하고 팀원 간 원활한 협업을 지원하기 위한 클라우드 기반 파일 관리 시스템을 구축한다. 
### 1.2.2 주요 기능 설명
| ID | 설명 |
| :--- | :--- |
| F01 | 파일 업로드/다운로드 (문서, 이미지, PDF 등 다양한 형식 지원) |
| F02 | 폴더 조직화 (폴더 생성, 이름 변경, 이동, 삭제 및 복구) |
| F03 | 파일 공유 및 권한 설정 (조회, 수정, 댓글, 외부 링크 공유) |
| F04 | 파일 검색 (이름, 날짜, 사용자, 유형 기준 검색) |
| F05 | 파일 버전 관리 (목록 확인, 다운로드, 이전 버전 복원) |

## 1.3 프로젝트 목표
- 조직 내 파일을 중앙에서 관리하여 최신 버전 유지
- 사용자 친화적인 웹 기반 업로드 및 다운로드 환경 제공 
- 보안을 유지하면서 효율적인 팀 간/외부 파일 공유 지원
  
## 1.4 용어 정의
| 용어 | 설명 |
| :--- | :--- |
| API | 응용 프로그램에서 사용할 수 있도록 제공하는 인터페이스 |
| 클라우드 | 인터넷을 통해 저장 공간과 컴퓨팅 자원을 제공하는 환경 |

# 2. 개발 계획

## 2.1 개발 절차 모형 
1. **방법론:** 요구사항의 빈번한 변경에 대응하기 위해 **Agile 방법 중 SCRUM 모델**을 채택한다.
2. **모델링:** 확장성과 유지보수성을 위해 **객체지향 방법론**을 사용한다.

## 2.2 WBS(Work Breakdown Structure)
- **요구분석:** 요구사항 정의, 기능 정의서 작성
- **설계:** 시스템 구조 설계, DB 설계, 상세 UI 설계 
- **코딩:** 서버 및 DB 구축, 주요 기능(업로드, 공유, 검색) 구현
- **테스트:** 단위 테스트, 통합 테스트, 매뉴얼 작성 

## 2.3 개발 내용
| 개발 활동 | 설명 |
| :--- | :--- |
| 요구사항 분석 | Customer Requirement Document 기반 분석서 작성 |
| Sprint | 1~4주 단위의 반복적인 기능 개발 및 피드백 |
| Scrum Meeting | Daily Scrum, Sprint Planning, Sprint Review, Retrospective 등을 각 Sprint마다 반복 수행 |

# 3. 팀 구성

## 3.1 팀 구조 
프로젝트는 **하이브리드 팀 구조(Hybrid Team Structure)**를 채택한다.  

PM을 중심으로 전체 방향성을 관리하는 중앙집중형 요소와,  
각 기능별 개발 파트가 자율적으로 작업을 수행하는 분산형 요소를 결합하여  
효율적인 의사결정과 유연한 개발 환경을 동시에 확보한다.

## 3.2 역할 및 책임 
| 역할 | 인원 | 책임(공동/개별) |
| :--- | :---: | :--- |
| 팀장(PM) | 1명 | 프로젝트 전반 진행 관리 및 검토 |
| 문서/DB | 2명 | 요구사항 문서화 및 DB 스키마 관리 |
| 웹 개발 | 3명 | 웹 브라우저 기반 인터페이스 및 공유 기능 개발 |
| QA | 1명 | 테스트 계획 수립 및 품질 검증 |

# 4. 품질 관리 

## 4.1 팀 미팅 계획
- **Sprint Planning:** 매주 월요일, 목표 및 백로그 계획 수립 
- **Daily Scrum:** 매일 오후 6시, 30분간 진행 상황 공유 
- **Sprint Review/Retrospective:** 스프린트 종료 시 회고 및 리뷰 

### 4.2 변경 사항 관리 방법 
기능 변경 시 팀원 전체의 합의를 거쳐야 하며, 변경 내용은 반드시 기록물로 남겨 프로젝트 관리자에게 보고한다. 

### 4.3 산출물 관리 방법 
- **명명 규칙:** [미니드라이브]문서명_YYMMDD_v버전 
- **저장 방법:** 소스 코드는 **GitHub**, 문서는 **공유 클라우드 서버**에 저장
