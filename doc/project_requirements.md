# 1. 서론

## 1.1 문서 목적 및 범위

본 문서는 조직 내에서 파일을 중앙 관리하고 팀원 간 원활한 협업을 지원하기 위한 **클라우드 파일 공유 시스템(미니 드라이브)**의 요구사항을 정의하는 문서이다.
본 문서에서는 기능적, 비기능적, 인터페이스 및 기타 개발 환경 요구사항을 구체적으로 기술한다.

---

## 1.2 대상 시스템 개요

### 1.2.1 프로젝트 정의

본 시스템은 조직 내 파편화된 파일 관리 문제를 해결하고 보안성을 강화하기 위한 **클라우드 기반 파일 저장 및 협업 시스템**이다.

---

### 1.2.2 주요 기능 설명

| ID  | 설명                                |
| :-- | :-------------------------------- |
| F01 | 파일 업로드/다운로드 (다양한 형식 지원)           |
| F02 | 폴더 조직화 (생성, 이름 변경, 이동, 삭제 및 복구)   |
| F03 | 파일 공유 및 권한 설정 (조회, 수정, 댓글, 링크 공유) |
| F04 | 파일 검색 (이름, 날짜, 사용자, 유형 기준)        |
| F05 | 파일 버전 관리 (목록 확인, 이전 버전 복원)        |

---

## 1.3 용어 정의

| 용어   | 설명                                      |
| :--- | :-------------------------------------- |
| API  | 응용 프로그램에서 기능을 제어할 수 있게 만든 인터페이스         |
| 클라우드 | 인터넷을 통해 데이터를 저장하고 관리하는 환경               |
| RBAC | 역할 기반 접근 제어 (Role-Based Access Control) |

# 2. 요구사항

## 2.1 기능적 요구사항

| ID     | 요구사항                                   | 우선순위 |
| :----- | :------------------------------------- | :--: |
| FR-001 | 사용자는 이메일과 비밀번호를 이용하여 회원가입 및 로그인을 수행한다. |   상  |
| FR-002 | 관리자는 사용자 계정을 생성, 삭제하고 권한을 부여한다.        |   상  |
| FR-003 | 관리자는 사용자별 저장 공간을 설정하고 관리한다.            |   중  |

---

### 2.1.2 파일 및 폴더 관리

| ID     | 요구사항                                 | 우선순위 |
| :----- | :----------------------------------- | :--: |
| FR-004 | 사용자는 웹 브라우저를 통해 파일을 업로드 및 다운로드한다.    |   상  |
| FR-005 | 사용자는 폴더를 생성하고 파일을 이동, 변경 및 삭제한다.     |   상  |
| FR-006 | 시스템은 삭제된 파일을 일정 기간 보관하여 복구 기능을 제공한다. |   중  |

---

### 2.1.3 파일 공유 및 버전 관리

| ID     | 요구사항                                  | 우선순위 |
| :----- | :------------------------------------ | :--: |

| FR-008 | 시스템은 만료 기간이 설정된 외부 공유 링크를 생성한다.       |   상  |
| FR-009 | 시스템은 파일 수정 시 이전 버전을 저장하고 복원 기능을 제공한다. |   상  |
