

# [미니 드라이브] 테스트 결과 보고서

## 1. 서론

### 1.1 문서 목적 및 범위

본 문서의 목적은 클라우드 파일 공유 시스템(미니 드라이브)에 대한 테스트를 수행하고 그 결과를 기록하는 데 있다.

본 문서에서는 프로젝트의 테스트 기준, 테스트 범위, 테스트 케이스 및 테스트 결과를 명시한다.

### 1.2 프로젝트 개요

미니 드라이브는 조직 내 파일을 중앙에서 관리하고 팀원 간 원활한 협업을 지원하기 위한 클라우드 기반 파일 관리 시스템이다.

본 시스템은 파일 업로드 및 다운로드, 폴더 관리, 파일 공유 및 권한 설정, 파일 검색, 파일 버전 관리 기능을 제공한다.

### 1.3 용어 정의

| 용어   | 설명                                     |
| ---- | -------------------------------------- |
| API  | 응용 프로그램에서 사용할 수 있도록 제공하는 인터페이스         |
| 클라우드 | 인터넷을 통해 저장 공간과 컴퓨팅 자원을 제공하는 환경         |
| RBAC | 역할 기반 접근 제어(Role-Based Access Control) |

# 2. 테스트 개요

## 2.1 테스트 범위

테스트는 시스템 내부 구조를 고려하지 않는 블랙박스 테스트를 기반으로 수행한다.

| 범위     | 내용                      |
| ------ | ----------------------- |
| 사용자 관리 | 회원가입 및 로그인 기능 확인        |
| 파일 관리  | 업로드, 다운로드 및 폴더 관리 기능 확인 |
| 공유 기능  | 권한 설정 및 외부 공유 기능 확인     |
| 검색 기능  | 파일 검색 및 필터링 기능 확인       |
| 버전 관리  | 파일 버전 저장 및 복원 기능 확인     |
| 성능     | 응답 시간 및 동시 사용자 처리 확인    |
| 보안     | 권한 없는 접근 차단 확인          |

## 2.2 테스트 항목 및 통과 기준

### 기능별 테스트 항목

| 기능별 테스트 항목 | 통과 기준                        |
| ---------- | ---------------------------- |
| 회원가입 기능    | 정상 형식의 이메일 및 비밀번호 입력 시 가입 성공 |
| 로그인 기능     | 등록된 계정 정보만 로그인 성공            |
| 파일 업로드 기능  | 지원 파일 형식 업로드 성공              |
| 파일 다운로드 기능 | 업로드된 파일 다운로드 성공              |
| 파일 검색 기능   | 검색 결과 정확히 출력                 |
| 파일 공유 기능   | 권한 설정 후 공유 성공                |
| 버전 관리 기능   | 이전 버전 복원 성공                  |

### 비기능별 테스트 항목

| 비기능별 테스트 항목 | 통과 기준                |
| ----------- | -------------------- |
| 브라우저 테스트    | Chrome, Edge에서 정상 동작 |
| 성능 테스트      | 응답 시간 3초 이하          |
| 보안 테스트      | 권한 없는 접근 차단          |
| 동시 사용자 테스트  | 200명 접속 시 정상 동작      |

# 3. 테스트 케이스

## 3.1 테스트 케이스 선정

테스트 케이스는 기능 요구사항과 비기능 요구사항을 기준으로 선정하였다.

본 프로젝트는 클라우드 파일 공유 시스템으로서 파일 업로드, 다운로드, 공유, 검색, 보안 기능을 핵심 기능으로 정의하고 해당 기능에 대한 테스트를 우선 수행하였다.

또한 블랙박스 테스트 기법을 적용하여 시스템 내부 구현이 아닌 사용자 관점에서 기능이 정상적으로 수행되는지 확인하는 데 중점을 두었다.

## 3.2 테스트 케이스 유형

본 프로젝트의 테스트 케이스 유형은 다음과 같다.

* 입력값 검증 테스트
* 기능 테스트
* 예외 처리 테스트
* 성능 테스트
* 보안 테스트
* 브라우저 호환성 테스트
